What is EAR Explorer?

EAR Explorer is an internal developer tool for reading and navigating the HighJump Advantage WMS application logic stored in the EAR database. It lets you search across every process, action, screen, SQL statement, and expression in the WMS — and trace exactly how execution flows between them.

🔍 Full-Text Process Search

Search by keyword across every object type in the WMS application logic simultaneously.

  • Process Search process names
  • Calculate Names and arithmetic expressions
  • Compare Names and operand values
  • Database Names and embedded SQL text
  • Dialog Screen names and field labels
  • Field / Record / Constant / Locale Names and values
  • Execute / Publish / Receive / Send / Report / User / List Step labels

Results are scoped to a specific Application (WA, MA, YA) and any combination of object types via the 🔎 Options dropdown. Searches run live against the WMS SQL Server.

📋 Process Step Viewer

Click any search result to see its full execution sequence in a structured grid:

  • Line number, label, action type, and step description
  • On Pass / On Fail branch targets per step
  • Click any action row to open an inline detail panel with full SQL, expression tree, dialog field layout, or comparison operands
  • Screen dialogs render an interactive grid showing the exact terminal screen layout (field positions, labels, prompt text) in 6×40 / 8×16 / 8×20 format

🔗 Caller & Call-Path Tracing

For any process, instantly find:

  • Direct callers — every process that calls this one (one hop)
  • All entry-point paths — reverse BFS from the process up to every device entry point, showing the full call chain
  • Action-level paths — trace a specific Calculate, Database, or Compare action back to its entry points

Results are displayed as a collapsible process list. Click any item to drill into its steps.

🗄️ Multi-Server Support

Switch between WMS environments without leaving the browser. The server selector in the header supports any of the configured SQL Server instances:

  • ArcadiaWHJSqlStage — WA staging warehouse
  • RetailRHjsqldev — Retail development
  • RetailRHjsqlStage — Retail staging

Every API call is scoped to the selected server; no page reload required.
